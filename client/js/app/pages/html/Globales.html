Avec javascript, toutes les variables déclarées en dehors d'une fonction seront des globales utilisables dans tous les scopes.

Plus précisément, toute nouvelle variable globale est donnée en propriété à l'objet du browser "window" (toutes les propriétés de window sont des "globales" et toutes les "globales" sont en fait des propriétés de window. ex: window.document, qu'on peut simplement utiliser avec la vrarible globale "document" (document.body, etc.).
Ainsi, déclarer var dojoConfig = {}; est l'équivalent de window.dojoConfig = {}; et dojoConfig pourra être utilisé tel quel dans tout le code à suivre et dans toutes les fonctions

Lorsque l'on charge Dojo sur une page web, nous avons directements accès à quelques globales minimales.
Depuis sa version 1.7, Dojo se veut "baseless", c'est à dire qu'au démarrage, aucune fonctionnalité n'est vraiment chargée, à l'exeption de celles qui permettent le chargement des autres.

Ces globales sont :

  • require - résultat du typeof :
  • define - résultat du typeof :
  • dojo, dijit, dojox qui servirons de namespace (c'est à dire de conteneurà pour les modules chargés. Notons qu'ils continnent aussi des éléments de rétro-compatibilité avec les versions antérieures à la 1.7, et des éléments de configuration.
  • Eventuellement l'objet dojoConfig si vous l'avez défini en tant que globale avant de charger Dojo, pour configurer Dojo.
